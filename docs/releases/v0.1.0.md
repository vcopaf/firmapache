# FirMapache v0.1.0

Primera version publica estable de FirMapache.

## Resumen

FirMapache v0.1.0 formaliza el firmador local de escritorio para JSON/JWS y
PDF/PAdES con soporte PKCS#11 y PKCS#12. Esta version esta pensada como base
estable para publicacion publica en GitHub, pruebas con tokens reales y
validacion de integraciones locales mediante `POST /sign`.

## Funcionalidades principales

- Servicio HTTPS local en `https://localhost:4637/`.
- Endpoint compatible `POST /sign` para JWS y PDF.
- Firma JWS compact RS256 compatible con `x5c`.
- Firma PDF `ETSI.CAdES.detached` con `/ByteRange`, `/Contents`, `/M` y
  metadatos basicos del diccionario de firma.
- Firma manual JSON/JWS y PDF desde la app.
- Firma manual multiarchivo con progreso, resumen y ZIP cuando hay multiples
  resultados.
- Soporte PKCS#11 para tokens fisicos, incluyendo ePass2003.
- Soporte PKCS#12/PFX para tokens virtuales de QA y desarrollo.
- Identidades de firma normalizadas.
- Autofirma configurable para entornos controlados.
- Cache de tokens/certificados y watcher PC/SC.
- Tray, segundo plano y ventana dedicada de firma.
- Validacion JWS, diagnostico estructural PDF y diagnostico exportable.

## Metadata de build

La aplicacion expone metadata centralizada:

- `APP_NAME`: FirMapache
- `APP_VERSION`: 0.1.0
- `BUILD_DATE`: fecha UTC generada al compilar
- `GIT_COMMIT`: commit corto generado al compilar
- `RELEASE_CHANNEL`: stable

Esta informacion aparece en el Dashboard, Diagnostico y endpoint
`GET /version`.

## Limitaciones conocidas

- PDF/PAdES no incluye TSA, LTV, OCSP ni CRL.
- El certificado HTTPS local es self-signed.
- Los tokens virtuales PKCS#12/PFX son recomendados para QA y desarrollo, no
  como reemplazo de tokens fisicos en produccion.
